## Documentation Best Practices

### When to Update

- **README.md**: On major version releases or significant feature additions
- **QUICK_START.md**: When features change significantly
- **TROUBLESHOOTING.md**: When new issues are discovered or resolved
- **Architecture docs**: When architecture changes

### When to Add New Docs

- New major features require documentation
- Complex workflows need step-by-step guides
- Known issues need troubleshooting entries

### When to Archive

- Session notes after project milestones
- Temporary fix documentation after fixes are verified
- Old analysis reports superseded by newer ones
